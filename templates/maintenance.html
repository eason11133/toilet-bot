Auto Verification 1.5.4 說明

這頁是給管理者定期維護使用者新增廁所資料用，不是給一般使用者看的查詢頁。

目前版本定位：大部分資料由系統自動通過；只有「名稱太籠統、嚴格重複、店家缺地址等真的需要人工判斷」才留下 pending。 所以看到很多 possible_duplicate_high 不代表資料壞掉，它只是相似資料提醒；真正需要處理的是 strict_duplicate 或 pending 佇列。

一、上方數字是什麼意思？

欄位意思
Approved系統判定低風險，可進入推薦資料池。
Pending需要管理者人工確認的資料，例如嚴格重複、名稱太籠統、店家缺地址。
Rejected明顯錯誤資料，例如座標非法、測試資料；目前規則很保守，通常會很少。
平均驗證分數系統自動驗證分數平均值。分數高不代表一定 approved；如果疑似重複，資料本身完整也可能 pending。

二、常見英文風險標籤

標籤中文意思處理建議
possible_duplicate_high高度相似提醒只是提醒附近有相似資料，不一定要處理。
strict_duplicate嚴格重複待審優先確認是否和既有資料重複，必要時合併或保留較完整的一筆。
possible_duplicate_medium中度重複提醒可抽查，不一定要每筆處理。
shop_access_unclear店家開放不明便利商店、超市、店家型廁所，缺少是否對外開放或入口說明。
missing_entrance_hint缺入口／位置提示室內或大型場域建議補「靠近哪裡、入口在哪」。
missing_floor_hint缺樓層資訊百貨、車站、學校、醫院、大樓類資料建議補樓層。
name_too_generic名稱太籠統例如只寫「公廁」「廁所」，建議改成可辨識名稱。
shop_missing_address店家缺地址店家型資料沒有地址時應人工確認。
spatial_outlier_candidate空間孤立提醒附近參考資料較少，先提醒，不直接判錯。
address_coordinate_mismatch地址座標可能不一致若之後開啟地址座標比對，座標和地址距離差太大會出現。

三、平常怎麼維護？

  • 平常只需要按 載入資料，看 待人工確認、嚴格重複待審、名稱太籠統。
  • 如果剛更新自動驗證規則，先按 預覽重新驗證，確認 approved / pending 比例合理。
  • 預覽合理後，再按 正式自動審核，系統才會真的寫回 Neon。
  • 正式審核後再按一次 載入資料，確認數字更新。

四、什麼時候不要按正式自動審核？

  • 如果預覽結果 pending 突然暴增，例如從十幾筆變成兩百多筆。
  • 如果 rejected 突然大量出現。
  • 如果錯誤訊息顯示某個函式不存在或 API 失敗。
